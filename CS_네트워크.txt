CS_네트워크

* OSI 7계층
- 네트워킹에 대한 표준을 7계층으로 나눈 것(통신 규칙)
- 상위계층에서 하위계층으로 데이터 이동됨.

 * 응용 계층
  - 사용자와 가장 밀접한 계층으로 인터페이스 역할
  - 응용 프로세스 간의 정보 교환을 담당
  - 전자메일, 인터넷, 동영상 플레이어 등

 * 표현 계층
  - 데이터 암호화 및 복호화
  - 송신자에서 온 데이터를 해석하기 위한 응용계층 데이터 부호화, 변화
  - 수신자에서 온 데이터의 압축을 풀 수 있는 방식 된 데이터 압축
 
 * 세션 계층
  - 통신 장치 간 상호작용 및 동기화 제공
  - 연결 세션에서 데이터 교환과 에러 발생 시 복구 관리

 * 전송 계층
  - 전송되는 단위 세그먼트
  - TCP, UDP
 - 종단간 신뢰성 있고 정확한 데이터 전송을 담당
  - 오류검출 및 흐름제어와 중복검사 수행

 * 네트워크 계층
  - 전송되는 단위 패킷
  - 라우팅 기능을 맡고 있는 계층으로 목적지까지 가장 안전하고 빠르게 데이터를 보낼 수 있음
  - 라우터, L3장치 사용

 * 데이터 링크 계층
  - 전송되는 단위 프레임
  - MAC 주소를 가지고 통신함
  - 브릿지나 스위치를 통해 맥주소를 가지고 물리계층에서 받은 정보 전달

 * 물리 계층
  - 전송되는 단위 비트
  - 주로 전기적, 기계적, 기능적인 특성을 이요해 통신 케이블로 데이터 전송
  - 전기적으로 On, Off 상태
  - 단지 데이터 전달만 할 뿐, 데이터 상태나 오류 확인 안함
  - 케이블, 리피터, 허브를 통해 데이터 전송

- OSI 7계층을 나눈 이유는?
통신이 일어나는 과정을 단계별로 파악하기 위함

*TCP, UDP
- TCP
신뢰성 높은 프로토콜(신뢰성 : 응답이 잘 오가는지)
3-way handshaking과 4-way handshaking 연결이 이루어짐
데이터 흐름제어 및 혼잡제어
1대1통신
패킷관리 필요 없음

- UDP
비연결형 프로토콜이며, 연결 없이 통신 가능
손상된 데이터를 재전송하지 않음
빠른 프로토콜로 스트리밍 서비스에서 주로 사용 
1 대 1 통신, 1 대 다 통신, 다 대 다 통신
패킷 관리 필요

* IP(Internet Protocol)
- 인터넷에서 데이터 전달 프로토콜(규약)
- 네트워크에서 각 호스트(컴퓨터)와 통신할 때 어디있는지 알아야 하기 위해 사용함
- 비신뢰성 : 보낸 데이터의 흐름에 관여하지 않아 전달 보장되지 않음
- 비연결성 : 패킷을 받을 대상이 연결 상태임을 확인하지 않고 전송함

* DNS(Domain Name Server)
- 도메인 주소를 IP 주소로 변환해주는 시스템
( ex. naver.com -> 192.168.xxx.xxx)

* 로드밸런서
- 서버의 부하를 분산시켜주는 시스템
 * L4
  - 전송계층
  - 4계층 이하의 정보를 가지고 로드 분산함
  - 포트 기반 부하 분산
  - Mac주소, IP 주소, 포트 정보를 가지고 트래픽 분산
 * L7 
  - 응용계층
  - 요청(URL) 기반 부하 분산
  - 패킷을 열어보면서 확인하기 때문에 비정상적인 트래픽(DDos) 방어 가능
